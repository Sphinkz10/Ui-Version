# Contributing to V-Login

First off, thank you for considering contributing to V-Login! It's people like you that make V-Login such a great tool.

## Where do I go from here?

If you've noticed a bug or have a feature request, please verify if an issue already exists. If not, feel free to open a new one.

## Bug Reports and Feature Requests

When opening an issue, please try to narrow down the problem to a reproducible state. Ideally, provide a list of steps to reproduce the issue, along with information regarding your environment (e.g., browser, OS, device).

## Pull Requests

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes (`npm test`).
5. Make sure your code lints (`npm run lint`).
6. Issue that pull request!

## Code Style

- Please follow the project's existing coding conventions.
- Use `npm run lint` before committing to catch any styling issues.

## Development Setup

1. Install dependencies: `npm install`
2. Run development server: `npm run dev`
3. We utilize Next.js / Vite in our tooling, depending on the active workspace. Ensure you build and verify any routing or API adjustments accordingly.

Thank you!
